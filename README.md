# Лабораторная работа на тему "Система управления доступом" по дисциплине "Моделирование безопасных компьютерных систем"

Система управления доступом пользователей к символам через графический интерфейс

Теория:

Модель управления доступом
Система реализует модель дискреционного управления доступом (DAC), где:
- Субъекты - пользователи системы
- Объекты - символы (буквы, цифры, специальные знаки)
- Права доступа - связи между субъектами и объектами

Принципы безопасности
- Аутентификация - проверка существования пользователя в БД
- Авторизация - контроль доступа только к разрешённым символам
- Изоляция данных - разделение прав через реляционную модель
  
О проекте

Система состоит из двух приложений на Go с GUI (Fyne):

Пользовательское приложение - для работы с разрешёнными символами

Админ-панель - для управления пользователями и правами доступа

Оба приложения взаимодействуют через единую базу данных SQLite, демонстрируя принципы многозвенной архитектуры.

Система может быть расширена для решения реальных задач контроля доступа в корпоративных средах.

Функциональность:

В ходе работы были написаны две программы: для пользователя и для 
администратора системы. Программа администратора представлена на 
рис. 1 и рис. 2.

Программа позволяет создавать и удалять субъекты и 
объекты как по одному, так и списком. Так же администратор имеет 
возможность выдать или удалить права на определенные объекты как 
одному, так и нескольким субъектам сразу.

Структура базы данных:

users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  
    name TEXT NOT NULL UNIQUE              
)

letters (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  
    char TEXT NOT NULL UNIQUE              
)

user_letters (
    user_id INTEGER,                      
    letter_id INTEGER,                    
    PRIMARY KEY (user_id, letter_id),     
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (letter_id) REFERENCES letters(id)
)
